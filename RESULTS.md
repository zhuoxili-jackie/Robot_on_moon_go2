# 验收标准与各代结果

> 验收用 `eval_policy.py`：对 5 个固定指令做**确定性回放**，量化指标对照阈值逐项 PASS/FAIL。
> 验收产物（`metrics.json` / 步态相位图 / 渲染蒙太奇 / gif）落在 `eval_out/<run>/`（不纳入版本库），
> 故每代关键数值在本文件以表格留存。**最新一代在最上。**

## 验收标准（THRESHOLDS）

5 个测试指令（`[vx, vy, yaw_rate]`，m/s & rad/s）：
`fwd_0.3 [0.3,0,0]`、`fwd_0.5 [0.5,0,0]`、`fwd_0.8 [0.8,0,0]`、`fwd_yaw [0.5,0,0.4]`、`lateral [0.4,-0.2,0]`。

| 指标 | 阈值 | 含义 |
|---|---|---|
| `no_fall` | 必须 True | 整个 episode 不跌倒 |
| `vx_track_mae` | ≤ 0.12 m/s | 前进速度跟踪平均绝对误差 |
| `vyaw_track_mae` | ≤ 0.25 rad/s | 偏航角速度跟踪误差 |
| `duty`（每足） | 0.25 – 0.85 | 触地占空比（trot ≈ 0.5） |
| `flight_frac` | ≤ 0.35 | 四足同时离地时间占比 |
| `force_sat_frac` | ≤ 0.10 | 力矩达 97% 上限的时间占比 |
| `action_sat_frac` | ≤ 0.50 | \|action\|>0.95 的占比（**占位值**，见 v1.1 注） |
| `pitch_std` / `roll_std` | ≤ 8.0° | 躯干姿态抖动 |

> 阈值为初版；`action_sat 0.50` 是占位值。**暂不放宽**——它当前 fail 正好反映「低趴 / 动作顶满」的真实待修问题。

## 训练时长与收敛（多 run 分析，2026-06-14）

对全部已完成 run 的 `rollout/ep_rew_mean` 曲线做收敛分析，回答「该训多少 M」。决定性证据是两个 5M run
（下表为达到各自最终 reward 的百分比）：

| 步数 | go2_baseline_5M | go2_5M_z40 | 边际增益/0.5M |
|---|---|---|---|
| 2.0M | 88.8% | 90.9% | ~+8% |
| 2.5M | 92.2% | 94.3% | ~+4% |
| 3.0M | 96.0% | 93.5% | +2~4% |
| **3.5M** | **97.8%** | **96.6%** | +2~3%（拐点） |
| 4.0M | 99.1% | 98.2% | +1.3~1.6% |
| 5.0M | 100% | 99.6% | +0.2~0.3% |

**结论**：~3.5M 后每 0.5M 提升 <2%（基本收敛）；最后 1.5M（3.5M→5M）总共仅 +2~3pp 却多烧 43% 算力，**5M 不值**。

**今后训练时长规范**：
- **配置筛选 / 横向对比**：**2.5M**（达 ~93%，~16 min；配置间排名约 2M 就定，如本轮 kp 扫）；
- **标准 / 默认（选定配置出正式结果）**：**3.5M**（达 ~97%，~22 min）—— `train_ppo.py` 的 `--total-timesteps`
  默认值已由 5M 改为 **3.5M**；
- **5M**：仅用于需榨干最后 ~2pp 的最终定版，常规不用。

## 各代结果（最新在上）

### v1.3 — 收窄 height 梯度修复后腿步态（go2_kp20_hnarrow_2p5M）★ 最佳 walker

- **日期**：2026-06-14 ｜ **run**：`go2_kp20_hnarrow_2p5M`（flat, 4 env, seed 1, CPU ~30 min, 2.5M 步）

**改动**：在 v1.1 基线（kp=20/kd=0.5、base_z=0.27、action_scale=[0.25,0.65,0.65]）上**只改 env 奖励**
（CHANGELOG 的 env② 约束，base_z 不动）：height 奖励 σ² 0.09→**0.04**（陡梯度，把 v1.2 用 base_z=0.4 得到的
步态修复效果留在 0.27 上复现）；height 权重 0.25→**0.4**；`min_base_height` 0.12→**0.18**（不许蹲低）；
新增 `_nonfoot_ground_contact_penalty()`——地形与非足端 geom 每接触一点扣 **−0.5**（封顶 4），逼脚底着地、
不拖小腿。`ep_rew_mean` 收敛到 ~1461（2.5M 时已达自身 final 的 99.2%）。

**验收：0/5（唯一不过项均为 `action_sat`，同基线；其余 8 项全过）**

**★ 核心成果**：v1.2 发现的「陡 height 梯度修复后腿走路」机制**成功搬到 base_z=0.27**——
① 后腿不再偷懒：触地频率四腿均匀（fwd_0.8 全 ~7/s；lateral 5.6/5.4/5.7/5.7），告别基线/kp25 的前快后慢；
② 不匍匐：站高稳定 0.205–0.208（kp25 当初蹲到 0.15）；③ 不跳跃：flight 前进 0.04–0.10；
④ 速度跟踪救回：vx_mae 0.04–0.065（kp25 的 fwd_0.8 曾 0.772 = 原地不动）；⑤ 零跌倒、零力矩饱和、姿态极稳。

**代价 / 待修**：`action_sat ~0.61–0.70`（>0.50 占位阈值）——陡梯度逼伸腿发力使动作幅度变大（v1.2 同现象 0.68–0.71），
是已知 trade-off、非真摔。**下一代方向**：提 kp（20→35）+ 缩 action_scale 压 action_sat，但须防 kp 偏硬重新引入不稳/跳跃。

### v1.2 — 诊断实验：base_z=0.4 隐式课程（go2_5M_z40）★ 发现性质

- **日期**：2026-06-14 ｜ **run**：`go2_5M_z40`（flat, 4 env, seed 1, CPU ~59 min, 5.005M 步）
- **交接文档对应**：v1.9

**改动**：在 v1.1 基线（kp=20/kd=0.5、action_scale=[0.25,0.65,0.65]、reward、PPO 全不变）基础上，
**单改** `go2.xml` 的 `stand` 关键帧 base_z：0.27→0.4（浮空 13cm）。`ep_rew_mean` 收敛到 ~1480。

**验收：0/5 通过**（每集唯一不过项均为 `action_sat`，同 v1.1 基线）

| 指令 | no_fall | vx_mae | vyaw_mae | duty(FR/FL/RR/RL) | flight | force_sat | action_sat | pitch/roll° | 站高 | 判定 |
|---|---|---|---|---|---|---|---|---|---|---|
| fwd_0.3 | ✓ | 0.016 | 0.027 | .72/.54/.43/.56 | 0.01 | 0.00 | 0.68 | 1.2/0.6 | 0.184 | ✗ action_sat |
| fwd_0.5 | ✓ | 0.021 | 0.023 | .66/.46/.35/.49 | 0.01 | 0.00 | 0.69 | 1.6/0.7 | 0.190 | ✗ action_sat |
| fwd_0.8 | ✓ | 0.039 | 0.029 | .61/.39/.31/.46 | 0.01 | 0.00 | 0.69 | 1.4/1.1 | 0.197 | ✗ action_sat |
| fwd_yaw | ✓ | 0.028 | 0.026 | .64/.35/.31/.53 | 0.01 | 0.00 | 0.71 | 1.5/1.2 | 0.191 | ✗ action_sat |
| lateral | ✓ | 0.017 | 0.033 | .69/.37/.34/.37 | 0.03 | 0.00 | 0.61 | 2.1/1.4 | 0.199 | ✗ action_sat |

**★ 核心发现**：基线 v1.1 的「关节走路」（后腿拖地、touchdowns 不对称，如 fwd_yaw 时 FL=8.4/s vs 其余~4.2/s）
**在本代被克服**——四条腿触地频率趋于均匀（fwd_yaw: [5.4,5.3,5.8,5.4]）。机理不是 spawn 高度本身，而是
`stand_height` 决定了高度奖励 `exp(−(z−h)²/0.09)` 的**梯度陡度**：0.27 弱梯度（dR/dz≈1.5）→ 策略偷懒 shuffling；
0.4 陡梯度（dR/dz≈2.9）→ 迫使伸腿探索 → 学到腿发力而非关节拖地。**「奖励梯度 = 步态质量的隐式课程」**。

**代价**：spawn 浮空 13cm（足端间隙 134mm，不实用）+ action_sat 68–71%（比基线高 ~15pp；策略拼命蹬腿但被
action_scale 上限卡住）+ pitch 略大 + vx=0.8 跟踪略差（0.039 vs 基线 0.025）。

**结论**：本代是**诊断实验**，非正式改进。它揭示了基线低趴/匍匐问题的根因分层——「关节走路」≠「整体低趴」，
前者是奖励梯度偷懒的产物，只需让高度奖励「更难满足」即可克服。**终版 spawn 仍用 0.27**，后续在 0.27 下通过
收窄 height sigma / 渐进课程退火等手段复现陡梯度效果，详见交接文档 §四 A2（v1.9 版）。

### v1.1 — 基线：Go2 裸机平地行走（5M PPO）

- **日期**：2026-06-14 ｜ **run**：`go2_baseline_5M`（flat, 4 env, seed 1, CPU ~62 min, 5.005M 步）

**改动**：Go2 原生 MJCF 力矩电机 → PD 位置伺服（kp=20 / kd=0.5，`ctrl=0`→stand，base_z=0.27）；
`Go2WalkEnv`（obs 52 / act 12，按关节名索引）；PPO 超参沿用 aliengo 蓝本。`ep_rew_mean` 收敛到 ~1530。

**验收：0/5 通过**（每集唯一不过项均为 `action_sat`）

|---|---|---|---|---|---|---|---|---|---|---|
| fwd_0.3 | ✓ | 0.017 | 0.027 | .71/.56/.44/.60 | 0.00 | 0.00 | 0.55 | 0.9/0.7 | 0.193 | ✗ action_sat |
| fwd_0.5 | ✓ | 0.015 | 0.023 | .59/.48/.42/.51 | 0.03 | 0.00 | 0.57 | 0.6/0.7 | 0.198 | ✗ action_sat |

| fwd_yaw | ✓ | 0.020 | 0.026 | .61/.47/.40/.57 | 0.09 | 0.00 | 0.58 | 1.0/0.9 | 0.194 | ✗ action_sat |

**结论**：稳定、**全向速度跟踪优秀**（vx_mae 0.015–0.027，转向 / 侧移都跟上）、零跌倒、零力矩饱和、步态规整小跑。
唯一短板：**站高仅 ~0.20 m（kp=20 偏软、明显低趴）** + **action_sat ≈ 0.55（动作顶满）**。
`0/5` 仅因 action_sat 卡阈，**非真摔** —— 本代**定为基线**。

**已知待修 → 下一代方向**（详细配方见上级项目 `../ref/CHANGELOG_from_go2_1M.md`）：
1. 先提 kp 20→**35**，`action_scale` 配套缩到 ≈[0.17,0.45,0.45]（抬高站姿、缓解饱和）；
2. 再加 env 约束（height 奖励收窄加重；先不动 base_z）；
3. 仍低趴 → kp→50 + 再缩 action_scale；4. 最后才动 base_z。
- ⚠️ `action_sat` 偏高 = 策略想要**更大**动作幅度，与「缩 action_scale」方向相反；提 kp 后位置误差应变小、所需幅度下降，整定时务必实测兼顾。
